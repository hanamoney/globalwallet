html ${menuName} ${CODE} 각 메뉴별 html

내부 작업시 최종 html 결과물에 추가된 이미지, css, js 등의 파일은 서버에 업로드 한 뒤 절대경로로 표기한다.

외부 작업시 최종 html 결과물의 이미지, css, js 등은 각각 /images, /css, /js 폴더에 위치시키며 상대경로로 표기한다.

네이밍 규칙

1. 파일명

- 영문소문자, 숫자, 언더스코어(_)로만 작성한다.

- 영문대문자, 숫자로 시작하지 않는다.

- 순서를 갖는 리스트 경우 첫번째 이름은 숫자를 생략할 수 있다.

- 네이밍 조합은 '형태_의미_순서_상태'를 기본 순서로 한다.

- 샘플이미지의 경우 맨 앞에 '@'를 붙인다.

2. id / class

- 스타일 적용을 위한 id, class는 언더스코어(_) 표기

- 스크립트 적용을 위한 id, class는 카멜케이스 표기

들여쓰기

코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 들여쓰기 규칙을 준수한다.

마크업의 중첩이 깊어질 때마다 자식 요소는 1탭 들여 쓰고, 1탭의 크기는 공백 2칸으로 설정한다.
