HTML 작업규칙

DTD

- html 문서는 HTML5를 기본으로 한다.

- DTD 선언시 앞에 공백이 오지 않도록 해야하며, 문서 최상단에 선언되어야 한다.

<!DOCTYPE html> <html lang="ko">

head

- meta, title, link, script 순서로 요소를 선언한다.

표준문법

- DTD를 제외한 모든 엘리먼트와 애트리뷰트는 소문자로 작성한다.

- 각 태그의 어트리뷰트 속성들에 값들은 항상 "(따옴표)를 사용한다.

- 특수기호는 Entity name을 사용하여 entity 코드로 변환한다. ( < , > , & 는 반드시 변환 )

주석

- 수정주석은 수정된 단락의 처음부분과 끝나는 부분에 주석을 적어 놓는다. 끝나는 부분의 주석은 // 처리를 한다.

- 한줄만 수정되는 간략한 경우는 해당태그 맨끝에 주석을 넣는다.

- 그룹을 감싸고 있는 객체를 구분하기 위하여 코드 그룹 간 1줄씩 빈 줄을 만드는 것은 허용한다.

<!-- YY.MM.DD 수정 --> <div> </div> <!-- // YY.MM.DD 수정 --> <p></p><!-- YY.MM.DD 수정 -->

속성 선언 순서

id, class를 제일 앞에 둔다.

data-* attribute는 마지막에 선언한다.

inline style 선언은 지양한다. 예외로 style 선언 시 가장 마지막에 선언한다.

input태그

<input type="text" id="" class="" title="" placeholder="" value="">

A태그

- href 속성의 값은 #none으로 한다. 앵커 기능이 있다면 해당 요소의 id값을 작성한다.

<a href="#none" class="" target="_blank"></a>

이미지태그

- 의미없는 이미지가 아닌 경우 alt 값 필수 (의미없는 경우 공백)

<img src="" class="" alt="">
