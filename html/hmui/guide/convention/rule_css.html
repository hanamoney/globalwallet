CSS 작업규칙

표준

- 속성 값 뒤에 세미콜른(;) 반드시 추가한다.

- 폰트의 선언, 배경 url, filter, content에 작은 따옴표('')를 사용한다. 그 외의 경우는 사용하지 않는다.

- .margin_top10 .pd20 등의 여백 클래스는 사용하지 않는다.

- CSS 주석 처리는 시작 주석을 작성하며 종료 주석은 작성하지 않는다.

속성 선언 순서

- 각각의 속성값에서 벤더픽스 추가 시 벤더 픽스 속성값을 먼저 선언한다.

- 형태, 흐름, 위치, 크기, 간격, 테두리, 배경, 폰트, 기타 순

- 속성은 top, right, bottom, left 시계방향으로 명시한다.

순서 의미 속성
1 형태 content, display, visibility
2 흐름 float, overfow
3 위치 position, z-index
4 크기 width, height
5 간격 margin, padding
6 테두리 border
7 배경 background
8 폰트 color, font-size, line-height, font-weight
9 기타 이 외의 속성은 순서 무관

약식 속성

색상코드

#fff, #000 줄여서 표기

margin, padding

약식 표기를 권장하며, 속성이 3개 이상 나열된다면 약식 표기로 기입한다.

background

background 약식 표기를 사용하며, 이후 속성이 변경될 경우 background 관련 속성을 사용하여 재선언 한다.

background-color, background-image, background-repeat, background-position, background-attachment 순

border

border 약식 표기를 사용하며, 이후 속성이 변경될 경우 border 관련 속성을 사용하여 재선언 한다.

width style color 순

font

font는 정식 표기를 권장한다.

단위

- 'rem' 을 기본으로 사용한다.

- border는 'px'로 사용한다.

- 해상도 비율에 따른 사이즈 설정이 필요한 경우 'vh, vw' 사용한다.

z-index

- 각 모듈별 안의 z-index의 설정값 사이 간격을 1로 한다.
